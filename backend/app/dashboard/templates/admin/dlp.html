Regex & Keyword Scanner Help
How it works

The regex scanner is the fastest tier (~0.1ms). It runs a set of built-in patterns plus any custom patterns you define against the full text of every request and response.

Built-in patterns (always active)

Adding custom patterns

Click Add Pattern and fill in:

  • Name — A human-readable label (e.g. "Vandal ID")
  • Regex — A Python-compatible regular expression. Use \b for word boundaries. The match is case-insensitive.
  • Category — The entity category name (used in severity rules). Use lowercase with spaces (e.g. "vandal id").
  • Severity — Default severity if no severity rule is defined for this category.

Keywords

Keywords are simple case-insensitive text matches. Enter one keyword per line. Good for catching phrases like "confidential", "classified", "FERPA", "HIPAA", or project-specific terms.

GLiNER NER Scanner Help
How it works

GLiNER is a neural named entity recognition (NER) model that detects sensitive entities in natural language text. Unlike regex, it understands context — it can detect names, phone numbers, and IDs even when they don't match a fixed pattern.

The model (urchade/gliner_multi_pii-v1, ~200MB) loads lazily on the first scan (~2-3 seconds). Subsequent scans take approximately 10ms on CPU.

Confidence threshold

The slider controls the minimum confidence score (0.0–1.0) for a detection to be reported. Lower values catch more entities but may produce false positives. Recommended: 0.5 for balanced detection, 0.7 for high precision.

Entity categories

Check the categories you want GLiNER to scan for. The built-in categories cover common PII types. You can also add custom categories by typing a name in the text field and clicking Add.

Custom categories work because GLiNER is a zero-shot model — it can detect entity types it wasn't explicitly trained on, as long as the category name is descriptive. Good custom categories:

  • vandal id — University of Idaho student IDs
  • grant number — Federal grant identifiers
  • diagnosis — Medical diagnoses (HIPAA)
  • grade — Student grades (FERPA)
  • medication — Drug/prescription names (HIPAA)
Dependencies

Requires the gliner Python package. The Docker image installs it automatically from pyproject.toml.

LLM Contextual Scanner Help
How it works

The LLM scanner sends the text to a dedicated language model for semantic analysis. Unlike regex or NER, the LLM understands context — it can detect that "my student John got a B+ in CS101" is a FERPA violation even though no single word is PII on its own.

The scanner self-routes through MindRouter's own API using a dedicated internal API key. This means it benefits from MindRouter's load balancing and model routing. The internal key's requests are automatically excluded from DLP scanning to prevent infinite loops.

Model selection

Enter any model name available in MindRouter. A small, fast model is ideal since DLP scans run in the background. Recommended: qwen/qwen3.5-4b or similar ~4B parameter model.

System prompt

The system prompt instructs the LLM what to look for and how to format its response. The default prompt asks for a JSON array of findings. You can customize it to focus on specific compliance regimes:

  • FERPA focus — Add: "Pay special attention to student records, grades, enrollment status, financial aid, and disciplinary records."
  • HIPAA focus — Add: "Focus on protected health information including diagnoses, treatments, medications, lab results, and patient identifiers."
  • CUI focus — Add: "Look for controlled unclassified information including export-controlled data, law enforcement sensitive data, and ITAR/EAR restricted content."
Performance

Typical latency is ~500ms depending on the model and text length. Text is truncated to 8,000 characters to limit token usage. The scan runs asynchronously after the response is delivered — it never delays user requests.

Severity Rules Help
How severity works

When DLP detects sensitive data, each finding has a category (e.g. "social security number", "email"). The severity rules map each category to a severity level. The highest severity among all findings in a single scan determines the alert level.

Severity levels
LevelMeaningTypical categories
MajorImmediate attention required. High-risk data exposure.SSN, credit card, bank account, passport, medical record
ModerateReview recommended. Potential compliance issue.Driver license, date of birth, student ID, vandal ID
MinorInformational. Low-risk but worth logging.Person name, email address, phone number
Adding rules

Type the category name (must match exactly what the scanner reports) and select a severity level. Categories from the GLiNER scanner match the checkbox labels in lowercase (e.g. "person", "social security number"). Custom regex pattern categories match the category field you defined.

If a detected category has no rule, it defaults to Moderate.
